Executive Summary

The HDK (Hyperdimensional Kernel Engine) is a zero-matrix-multiply text classifier implemented on an AMD Artix-7 XC7A200T FPGA. Unlike traditional neural network accelerators that rely on thousands of DSP slices and MAC operations, HDK uses only LUT logic — random binary projection followed by L-BFGS linear readout — achieving 71.09% accuracy on 20 Newsgroups with zero DSPs, zero BRAMs, and zero matrix multiplications.

This document presents measured results from Vivado 2025.2 synthesis and 10-fold cross-validated accuracy benchmarks, comparing HDK FPGA against commercial alternatives including the Google Edge TPU.

Inference Pipeline

The HDK classifier processes each input through five stages:

  1. Host → UART: HD vector (20K bits) received via UART at 10 Mbaud
  2. XOR Array: Input vector XOR'd with each of 20 class prototypes in parallel
  3. POPCOUNT Trees: 20 parallel Hamming distance counters (15-level pipeline)
  4. Argmin: Sequential comparator finds minimum distance → predicted class
  5. UART → Host: Class ID + confidence sent back

Conclusion

The HDK FPGA accelerator demonstrates that high-accuracy text classification can be achieved without DSPs, BRAMs, or matrix multiplications. By leveraging random binary projection and L-BFGS linear readout — the Random Kitchen Sinks framework — the HDK achieves statistical parity with sklearn Logistic Regression (71.09% accuracy, McNemar p > 0.05) while consuming only 52% of an Artix-7 FPGA.

Compared to the Edge TPU, HDK FPGA delivers 10× higher throughput, 21% higher accuracy, zero DSPs, and identical hardware cost ($150). These advantages are supported by measured Vivado synthesis data and 10-fold cross-validated accuracy benchmarks.
